# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.306.x | :white_check_mark: |
| 1.0.292.x | :white_check_mark: |
| < 1.0.292 | :x:                |

Only the latest version receives security updates. We recommend always updating to the newest release.

## Reporting a Vulnerability

If you discover a security vulnerability in this plugin, please report it responsibly:

1. **Use GitHub's Private Vulnerability Reporting** (preferred)
   - Go to the [Security tab](https://github.com/K3ntas/jellyfin-plugin-ratings/security)
   - Click "Report a vulnerability"

2. **Or create a private issue**
   - Email details to the repository owner

**Please include:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

**Response time:**
- Initial response: within 48 hours
- Status update: within 7 days
- Fix timeline: depends on severity (critical: ASAP, high: 1-2 weeks, medium/low: next release)

**Please do NOT:**
- Disclose the vulnerability publicly before it's fixed
- Exploit the vulnerability beyond proof-of-concept

We appreciate responsible disclosure and will credit reporters in the release notes (unless you prefer to remain anonymous).
